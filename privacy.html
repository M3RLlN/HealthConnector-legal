Privacy Policy — HealthConnector

Last updated: 25 April 2026 · Version 1.0

1. About this document

This Privacy Policy describes how HealthConnector processes personal data. HealthConnector is a non-commercial Bachelor thesis project at Hochschule RheinMain, Germany. The application is being designed and developed by a single student for that student's own personal use only. There are no other users.

2. What HealthConnector is

HealthConnector is a planned web application that will run entirely on the developer's own personal computer in Docker containers (a .NET backend, a PostgreSQL database, a Vue 3 frontend and Grafana for visualisation). Its purpose is to read the developer's own personal fitness and health data from third-party providers (currently the Huawei Health Service Kit and the Google Health API) and to keep a private local copy of that data on the developer's own computer.

3. Controller

The controller in the sense of Art. 4(7) GDPR is the developer of HealthConnector. For any privacy-related question please contact:

  • Email: marvin.fischer.inbox@proton.me

4. Data processed

HealthConnector only processes data that belongs to the developer themselves. Specifically, the application reads the following categories of data through the official APIs of the respective providers, after the developer has explicitly granted access via OAuth 2.0:

  • Activity data (e.g. steps, distance, calories, active minutes)
  • Cardiovascular and vital-sign data (e.g. heart rate, blood pressure, SpO2, body temperature)
  • Body-composition and wellbeing data (e.g. body fat, sleep, stress level)
  • Profile attributes needed for correct calculations (height, weight, gender, date of birth)
  • Device metadata (device name, model, serial number) used to label records with their source device

No data of any other person is processed. HealthConnector does not collect data from end users because there are no end users other than the developer.

5. Purposes of processing

  • Personal long-term self-tracking on the developer's own computer.
  • Academic research: the data forms the evaluation dataset of the developer's Bachelor thesis on multi-vendor personal health-data aggregation.

The data will not be used for medical diagnosis, medical treatment, medical advice, profiling, automated decision-making, advertising, or any commercial purpose.

6. Legal basis

The legal basis for processing the developer's own data is Art. 6(1)(a) GDPR (consent given by the data subject — who is the developer themselves) and, where applicable, Art. 9(2)(a) GDPR for the explicit consent to process health data.

7. Storage and security

  • All data is stored exclusively in a local PostgreSQL database running in a Docker container on the developer's own personal computer.
  • No data is uploaded to any cloud, hosted database, or external server operated by HealthConnector.
  • OAuth access tokens are stored encrypted and only locally; they are used solely to authenticate read requests against the provider APIs.
  • The developer's computer is protected by full-disk encryption and an OS-level user account password.

8. Sharing with third parties

HealthConnector does not share any data with any third party. The only outbound network traffic is the read-only API calls to the provider endpoints (Huawei Health Service Kit, Google Health API) needed to fetch the developer's own data.

9. Retention

Data is retained for as long as it is useful for the developer's personal records and the Bachelor thesis. The developer can delete the local PostgreSQL database (or any subset of it) at any time, which removes the data permanently.

10. Provider privacy policies

When data is read from a provider, that provider's own privacy policy applies in addition to this document:

  • Huawei: https://consumer.huawei.com/en/privacy/privacy-policy/
  • Google: https://policies.google.com/privacy

11. Rights of the data subject

Because the only data subject is the developer themselves, the GDPR rights (access, rectification, erasure, restriction, portability, objection — Art. 15–21 GDPR) are exercised by the developer directly through full local control over the database. Authorisations granted to HealthConnector against the provider APIs can be revoked at any time in the respective provider's account settings (Huawei ID privacy centre / Google Account permissions).

12. Changes

This Privacy Policy may be updated as the project evolves. The current version, including its date, is always available at the URL of this page.
